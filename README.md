# برنامج تحميل المقاطع — النسخة الاحترافية

تحديث كامل للشكل والبنية، بناءً على طلبك. هذا ملخص لكل شي تغيّر ووين تلقاه بالكود.

## الملفات

```
app.py           نقطة الدخول — الروابط (routes) وربط كل شي ببعض
config.py        كل الإعدادات القابلة للتعديل بمكان واحد
auth.py          نظام الدخول بجلسة (Session) بدل نافذة المتصفح الافتراضية
downloader.py    كل التعامل مع yt-dlp: جلب المعلومات، التحميل، إعادة المحاولة، الكوكيز
stats.py         تتبع الإحصائيات (عدد التحميلات، الحجم، الفاشل...)
templates/
  login.html     صفحة تسجيل الدخول
  index.html     الواجهة الرئيسية (تبويبات: تحميل / السجل / الإحصائيات / الإعدادات)
static/
  style.css      تصميم Glass + Dark/Light Mode + الأنيميشن
  app.js         كل منطق الواجهة (تبويبات، Toasts، رفع الكوكيز...)
  logo.svg       الشعار
```

## أهم التغييرات في الواجهة

- **تصميم Glass + أنيميشن**: بطاقات شفافة بتأثير ضبابي (backdrop-blur)، تدرّجات لونية، حركة عند التحميل والتفاعل.
- **متجاوب**: يشتغل بشكل مرتب على الجوال والكمبيوتر بدون تعديل إضافي.
- **الوضع الداكن/الفاتح**: زر تبديل بالأعلى، يُحفظ اختيارك بالمتصفح.
- **Toast Notifications**: أي رسالة نجاح أو خطأ تطلع كإشعار صغير أسفل الشاشة بدل `alert()`.
- **سجل التحميلات (تبويب "السجل")**: قائمة بكل ملف حمّلته مع حجمه، وزر حذف مباشر لكل ملف.
- **لوحة إحصائيات (تبويب "الإحصائيات")**: عدد التحميلات الكلي، فيديو مقابل صوت، الحجم الإجمالي، وعدد المحاولات الفاشلة.
- **معاينة المقطع**: تظهر صورة مصغّرة وعنوان ومدة المقطع فور جلب المعلومات، قبل ما تحمّله.

## حل مشكلة يوتيوب ("ما يقبل التنزيل")

السبب الشائع إن يوتيوب يطلب تأكيد "أنك مو بوت" لبعض الطلبات، خصوصًا من عناوين IP تبع الاستضافات. الحل المضاف الآن في **تبويب "الإعدادات"**:

1. سجّل دخول في يوتيوب على متصفحك العادي.
2. صدّر الكوكيز بصيغة `cookies.txt` (Netscape format) — أسهل طريقة إضافة إضافة متصفح مجانية زي "Get cookies.txt LOCALLY" لكروم.
3. من تبويب "الإعدادات" بالموقع، ارفع الملف.
4. من هذي اللحظة، كل تحميل يستخدم هذي الكوكيز تلقائيًا (بدون أي إعداد إضافي)، وهذا يحل أغلب حالات الحظر.

⚠️ ملاحظة: هذي الكوكيز مرتبطة بحسابك الشخصي في يوتيوب — لا ترفع الموقع لأي شخص غيرك وأنت حاطّ كوكيزك، وحافظ على كلمة مرور الموقع مفعّلة دائمًا.

## تحسينات الخلفية (Backend)

- **إعادة المحاولة تلقائيًا (Retry)**: أي تحميل يفشل لسبب مؤقت (شبكة، حظر مؤقت) يُعاد تلقائيًا حتى 3 مرات مع فاصل زمني متزايد، قبل ما يُعتبر فشل نهائي.
- **تنظيف تلقائي للملفات المؤقتة**: أي ملف عمره أكثر من 6 ساعات (قابل للتعديل عبر `FILE_MAX_AGE_HOURS`) يُحذف تلقائيًا كل نصف ساعة، حتى لا تمتلئ مساحة التخزين المؤقتة على الاستضافة.
- **رسائل خطأ مفهومة**: بدل رسالة yt-dlp التقنية الخام، الموقع يترجمها لرسائل عربية واضحة (مثلًا: "يوتيوب طلب تأكيد أنك لست بوت... ارفع كوكيز").
- **كود مقسّم ومنظّم**: كل مسؤولية بملف منفصل بدل ملف واحد ضخم.

## تسجيل الدخول والحماية

- بدل نافذة المتصفح الافتراضية، صار فيه **صفحة دخول حقيقية** (`login.html`).
- الجلسة محمية بكوكي موقّع بمفتاح سري (`SECRET_KEY`) — يُفضّل ضبطه كمتغيّر بيئة ثابت عند النشر (وإلا يتغيّر تلقائيًا كل إعادة تشغيل ويطلب دخول جديد من الكل).
- حماية بسيطة من محاولات التخمين المتكررة لكلمة المرور (حظر مؤقت بعد 8 محاولات فاشلة خلال 5 دقائق).

## متغيرات البيئة (تُضبط من لوحة الاستضافة)

| المتغيّر | الوصف | إلزامي؟ |
|---|---|---|
| `APP_PASSWORD` | كلمة مرور الدخول للموقع | يُفضّل بشدة عند النشر العام |
| `SECRET_KEY` | مفتاح تشفير الجلسة (أي نص عشوائي طويل) | مستحسن، وإلا تنقطع الجلسات عند كل إعادة تشغيل |
| `DOWNLOAD_DIR` | مسار التخزين المؤقت | اختياري (افتراضي `/tmp/downloads`) |
| `FILE_MAX_AGE_HOURS` | عمر الملف قبل الحذف التلقائي بالساعات | اختياري (افتراضي 6) |
| `DOWNLOAD_RETRIES` | عدد محاولات إعادة التحميل | اختياري (افتراضي 3) |

## API (لو تبي تستخدمه من أداة ثانية)

كل نقاط الدخول تحت `/api/` وتتطلب جلسة دخول صالحة (نفس كوكي تسجيل الدخول بالمتصفح):

- `POST /api/info` — `{url}` → معلومات المقطع
- `POST /api/download` — `{url, file_type, with_audio, quality, use_range, start_sec, end_sec}` → `{job_id}`
- `GET /api/progress/<job_id>` → حالة التحميل
- `GET /api/files` → قائمة الملفات المحمّلة
- `DELETE /api/files/<name>` → حذف ملف
- `GET /api/stats` → الإحصائيات
- `POST /api/cookies` (multipart, حقل `cookies`) → رفع ملف كوكيز
- `DELETE /api/cookies` → إزالة ملف الكوكيز
- `GET /api/cookies/status` → هل فيه كوكيز مرفوعة

## التشغيل محليًا

```
pip install -r requirements.txt
python app.py
```
افتح `http://127.0.0.1:5000` من المتصفح (مو فتح ملف HTML مباشرة).

## النشر على استضافة (نفس خطوات Render السابقة، بدون تغيير)

1. ارفع مجلد المشروع كامل على GitHub.
2. Render → New → Web Service → اختر المستودع (يكتشف `Dockerfile` تلقائيًا، فيه FFmpeg جاهز).
3. أضف متغيرات البيئة `APP_PASSWORD` و`SECRET_KEY` من تبويب Environment.
4. Deploy، وانتظر الرابط.
5. افتح الرابط، بيوديك لصفحة تسجيل الدخول الجديدة.

## ملاحظات مهمة (تبقى كما هي)

- شروط استخدام يوتيوب/تيك توك/إنستقرام وغيرها تمنع صراحة التحميل عبر أدوات خارجية — خلّه لاستخدامك الشخصي بكلمة مرور، مو أداة عامة.
- التخزين مؤقت (ephemeral) على أغلب الاستضافات المجانية، والتنظيف التلقائي الآن يحذف الملفات القديمة على أي حال.
